Customer In-Shop (Step-by-Step)

  1. Open /customer/appointments/new.
  2. Select shop.
  3. Select visit type in-shop.
  4. Select service.
  5. Pick date/time and enter vehicle/details/media.
  6. Submit -> POST /api/appointments.
  7. API creates appointment and linked workOrder status pending.
  8. Customer is routed to /customer/appointments.
  9. Shop progresses workOrder via /api/workorders/[id] updates.
  10. When status becomes waiting-for-payment, customer can pay from workorder detail or payments page.
  11. Stripe webhook closes workOrder and marks paid.

Customer Road-Call (Step-by-Step)

  1. Open /customer/appointments/new.
  2. Select shop.
  3. Select visit type road-call.
  4. Select service and provide roadside notes/media.
  5. Submit -> POST /api/appointments.
  6. API creates workOrder with serviceLocation road-call and location payload.
  7. Shop receives work in queue and dispatches/updates status.
  8. Work reaches waiting-for-payment.
  9. Customer pays via Stripe checkout.
  10. Webhook marks closed and paid; customer sees closed state.

Shop Employee In-Shop (Step-by-Step)

  1. Open /workorders/inshop.
  2. Redirects to /shop/new-inshop-job.
  3. Fill customer, vehicle, services, and notes.
  4. Submit -> POST /api/workorders.
  5. workOrder created in pending status.
  6. Shop assigns to bay/tech via /shop/home and PUT /api/workorders/[id].
  7. Estimate is added; status transitions to waiting-for-payment.
  8. Customer pays via checkout.
  9. Webhook closes and marks paid; invoice and receipt paths available.
