🚩 Feature Flags — defined in subscription.ts vs. enforced in subscription-access.ts
gpsVerification
Growth+
Flag exists. Route /tech/share-location exists. They are NOT connected. Starter plan users can share GPS for free right now.
breakTracking
Growth+
Marketed as a paid Growth feature. No route in subscription-access.ts enforces it. Break UI is silently accessible on all plans including Starter.
realTimeDashboards
Growth+
/manager/overview and /shop/calendar are ungated. Starter users see real-time data that's marketed as a Growth upgrade.
budgetTracking
Professional+
Defined as Professional+ but no standalone route is locked by this flag. If budget UI is embedded in payroll pages (which are gated) it may be OK — but needs verification.
whiteLabel
Enterprise only
/shop/branding is currently gated by advancedReporting (Professional+), not whiteLabel (Enterprise). Shops on Professional can rebrand — 2 tiers early.
messaging / inventory / payroll / advancedReporting / multiRoleUsers / photoCapture / multiShopManagement / revenueAnalytics / customIntegrations / apiAccess
✔ These 10 flags are actively used in subscription-access.ts and correctly gate routes.
prioritySupport / slaGuarantees / dedicatedSupport / unlimitedShops
✔ Service-level / capacity flags — no UI route enforcement needed. OK as-is.
